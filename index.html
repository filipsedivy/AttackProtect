Welcome on page project AttackProtect®.

This class provides a very effective against XSS and SQL Injection attacks. Since this class protects global variables $_GET and $_POST, so it is not necessary to further protect these variables in the code. After the initialization of protection, just as well as access to global variables using $_GET['test'] or $_POST['test'] where these variables are already protected.

Basic initialization

require_once 'src/AttackProtect.php';
use FS\AttackProtect;

Automatically protect

In the case of the use of automatic protection, are all requests $_GET and $_POST protected against XSS, thus entering the HTML code in a variable. 

AttackProtect::protect();

Advanced settings

If you need a variable to protect against SQL Injection attacks, i.e., a database, you can do additional settings. 

AttackProtect::protect(array(
    'number' => AttackProtect::SQL,
    'string' => AttackProtect::Disable
));

Now, therefore, the variable number will be protected against SQL Injection, so far for the variable string was completely disabled and protection for the other variables will be further protection against XSS. We can also set up for one variable and more protections, therefore, if we want to protect it against the Insert SQL Injection and also cast it to a number we can make use of the field. 

AttackProtect::protect(array(
    'text' => [AttackProtect::Input, AttackProtect::Number]
));

Default application setting

In the event that it is necessary to influence the State of the application, it is possible to override the default behavior. It is possible to influence by using thestatic variables. 

AttackProtect::$defaultProtect = AttackProtect::SQL;

These settings override the default privacy settings for all of the variables that are not particularly defined from XSS on SQL Injection. We can also use the field for setting, then combine the more settings at the same time.

Default setting

Variable Description Default value
$defaultProtect Default protect for none defined items in list AttackProtect::SQL

Types protect

Protect Description
AttackProtect::SQL Protect against SQL Injection
AttackProtect::Input Protect against XSS
AttackProtect::All Protect against SQL Injection and XSS
AttackProtect::Disable Disable protection
AttackProtect::Number If the entry is the number of the cast it otherwise the output is null
