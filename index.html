Vítejte na stránkách projektu AttackProtect®.

Tato třída poskytuje velmi účinnou proti napadením XSS a SQL Injection. Jelikož tato třída plošně chrání globální proměnné $_GET a $_POST, tak že není nutné tyto proměnné chránit dále v kódu. Po inicilizaci protekce, stačí jen přistupovat stejně jako ke globálním proměnným tedy pomocí $_GET['test'] nebo $_POST['test'] kde tyto proměnné již jsou ochráněny.

Základní inicializace

require_once 'src/AttackProtect.php';
use FS\AttackProtect;

Automatická ochrana

V případě využití automatické ochrany, jsou všechny requesty $_GET a $_POST chráněny proti XSS, tedy zanesení HTML kódu do proměnné. 

AttackProtect::protect();

Nastavení chování

Pokud potřebujete nějaké proměnné ochránit proti SQL Injection, tedy napadením databáze, můžete tak učinnit dodatečným nastavením. 

AttackProtect::protect(array(
    'cislo' => AttackProtect::SQL,
    'text' => AttackProtect::Disable
));

Nyní tedy, proměnná cislo bude chráněná proti SQL Injection, zatím u proměnné text byla ochrana úplně vypnuta a u ostatních proměnných bude dále ochrana proti XSS. Dále můžeme pro jednu proměnnou nastavit i více ochran, tedy pokud jej chceme ochránit proti vložení SQL Injection a zároveň přetypovat jej na číslo můžeme učinit pomocí pole. 

AttackProtect::protect(array(
    'text' => [AttackProtect::Input, AttackProtect::Number]
));

Výchozí chování aplikace

V případě že je nutné ovlivnit nějaký stav aplikace, je možné přepsat jeho výchozí chování. Je možné jej ovlivnit pomocí statických proměnných. 

AttackProtect::$defaultProtect = AttackProtect::SQL;

Tímto nastavením bylo přepsáno výchozí nastavení ochrany pro všechny proměnné které nejsou zvláště definované z XSS na SQL Injection. Můžeme i použít pole pro nastavení, tedy kombinovat více nastavení zároveň.

Výchozí nastavení

Proměnná Popis Výchozí hodnota
$defaultProtect Výchozí ochrana pro nedefinované položky v seznamu AttackProtect::SQL

Typy ochran

Ochrana Popis
AttackProtect::SQL Ochrana proti SQL Injection
AttackProtect::Input Ochrana proti XSS
AttackProtect::All Ochrana proti SQL Injection a XSS zároveň
AttackProtect::Disable Vypnutí ochrany
AttackProtect::Number Pokud vstupem je číslo přetypuje jej jinak výstupem je null
